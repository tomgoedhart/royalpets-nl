### Codebase Patterns

#### Testing Patterns
- Use jest.mock() for mocking dependencies
- Mock modules with selective imports: jest.mock("@/lib/prompts", () => ({ ...jest.requireActual("@/lib/prompts"), fn: jest.fn() }))
- Polyfill Response, Request, Headers, AbortSignal for API route tests in jest.setup.ts
- Mock NextResponse with static json() and redirect() methods
- Use --forceExit flag for jest to prevent hanging

#### API Route Patterns
- Use NextRequest from next/server for typed request handling
- Return NextResponse.json() with proper status codes
- Include rate limit headers on responses
- Use try-catch blocks with specific error handling
- Update database records on both success and failure

#### OpenAI Integration Patterns
- Configure OpenAI client with maxRetries and timeout
- Use exponential backoff with jitter for retries
- Don't retry on 4xx errors except rate limits (429)
- Generate variations with slightly different prompts for diversity

#### R2 Integration Patterns
- Use generateFileKey() for unique filenames
- Set isTemporary: true for 24h expiry on preview images
- Include metadata for tracking (portraitId, costumeId, etc.)

#### React Hook Patterns
- Use refs to store previous values for retry functionality
- Cleanup abort controllers and timeouts in useEffect cleanup
- Use renderHook from @testing-library/react for hook tests
- Separate polling logic from state management

#### Component State Patterns
- Use localStorage for session recovery across page refreshes
- Implement proper cleanup on unmount (cancelAnimationFrame, abort controllers)
- Use requestAnimationFrame for smooth progress animations
- Status messages as array of objects with duration for rotating display
- Wrap pages using useSearchParams in Suspense boundary with loading fallback
- Use aria-pressed for toggle/select buttons to indicate selected state

#### Zustand Store Patterns
- Use persist middleware from zustand/middleware for localStorage persistence
- Use createJSONStorage(() => localStorage) for SSR compatibility
- Define CreationState and CreationActions interfaces separately for type safety
- Use partialize option to select which state to persist
- Implement canNavigateToStep() for navigation guards between steps
- Use resetSession() to clear both state and localStorage
- Prefer external props over internal state for shared selections
- Helper functions for step names/descriptions keep UI text organized
